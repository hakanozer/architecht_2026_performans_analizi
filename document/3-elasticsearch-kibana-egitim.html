🔍 Elasticsearch Temelleri

Temel Kavramlar

  • Index: İlişkisel veritabanındaki tablo karşılığı. Loglar bir index'e yazılır (örn: restapi-logs-2026.04.12)
  • Document: Bir log kaydı. JSON formatında saklanır.
  • Mapping: Document'taki alanların veri tiplerini tanımlar (string, date, integer vb.)
  • Shard: Index'in parçalanmış alt bölümü. Paralel okuma/yazma sağlar.
  • Replica: Shard kopyası. Yüksek erişilebilirlik (HA) sağlar.

Elasticsearch Sağlık Kontrolü

# Cluster sağlık durumu
curl http://localhost:9200/_cluster/health?pretty

# İndeksleri listele
curl http://localhost:9200/_cat/indices?v

# Belirli bir index'in mapping'i
curl http://localhost:9200/restapi-logs/_mapping?pretty

# Basit arama
curl "http://localhost:9200/restapi-logs/_search?q=level:Error&pretty"

⚙️ .NET Core'dan Elasticsearch'e Log Gönderme (Serilog)

Adım 1: NuGet Paketleri

# Serilog paketleri
dotnet add package Serilog.AspNetCore
dotnet add package Serilog.Sinks.Elasticsearch
dotnet add package Serilog.Enrichers.Environment
dotnet add package Serilog.Enrichers.Process
dotnet add package Serilog.Enrichers.Thread
dotnet add package Serilog.Formatting.Elasticsearch

Adım 2: Program.cs'de Serilog Yapılandırması

using Serilog;
using Serilog.Sinks.Elasticsearch;
using Serilog.Formatting.Elasticsearch;

// Serilog yapılandırması
Log.Logger = new LoggerConfiguration()
    .MinimumLevel.Information()
    .MinimumLevel.Override("Microsoft", Serilog.Events.LogEventLevel.Warning)
    .MinimumLevel.Override("System", Serilog.Events.LogEventLevel.Warning)
    .Enrich.FromLogContext()
    .Enrich.WithEnvironmentName()
    .Enrich.WithMachineName()
    .Enrich.WithProcessId()
    .Enrich.WithThreadId()
    // Konsola yaz
    .WriteTo.Console()
    // Elasticsearch'e yaz
    .WriteTo.Elasticsearch(new ElasticsearchSinkOptions(
        new Uri("http://localhost:9200"))
    {
        // Her gün yeni index oluştur
        IndexFormat = $"restapi-logs-{DateTime.UtcNow:yyyy.MM.dd}",
        AutoRegisterTemplate = true,
        AutoRegisterTemplateVersion = AutoRegisterTemplateVersion.ESv8,
        // JSON formatı
        CustomFormatter = new ElasticsearchJsonFormatter(),
        // Batch ayarları
        BatchPostingLimit = 50,
        Period = TimeSpan.FromSeconds(2),
        // Hata durumunda dosyaya yaz
        BufferBaseFilename = "./logs/elastic-buffer",
        BufferFileSizeLimitBytes = 5242880, // 5MB
    })
    .CreateLogger();

var builder = WebApplication.CreateBuilder(args);

// Serilog'u host'a bağla
builder.Host.UseSerilog();

// ...

var app = builder.Build();

// Request loglama middleware
app.UseSerilogRequestLogging(options =>
{
    options.MessageTemplate =
        "HTTP {RequestMethod} {RequestPath} responded {StatusCode} in {Elapsed:0.0000} ms";
    options.EnrichDiagnosticContext = (diagnosticContext, httpContext) =>
    {
        diagnosticContext.Set("RequestHost", httpContext.Request.Host.Value);
        diagnosticContext.Set("UserAgent", httpContext.Request.Headers["User-Agent"].ToString());
    };
});
💡 İpucu: Serilog ile Elasticsearch'e doğrudan yazım yapılır. Logstash aracılığıyla da gönderilebilir, ancak doğrudan yazım daha basit ve düşük gecikmeli bir çözümdür.

📝 Structured Logging (Yapısal Loglama)

Neden Structured Logging?

Geleneksel string bazlı loglama yerine, yapısal (key-value) loglama kullanmak Elasticsearch'te arama ve filtrelemeyi dramatik şekilde kolaylaştırır.

❌ Yanlış (Unstructured): 
_logger.LogInformation("User " + userId + " created product " + productName);
✅ Doğru (Structured): 
_logger.LogInformation("User {UserId} created product {ProductName} with price {Price}",
    userId, productName, price);

Controller'da Structured Logging Kullanımı

[ApiController]
[Route("api/[controller]")]
public class ProductController : ControllerBase
{
    private readonly ILogger<ProductController> _logger;
    private readonly ApplicationDbContext _context;

    public ProductController(
        ApplicationDbContext context,
        ILogger<ProductController> logger)
    {
        _context = context;
        _logger = logger;
    }

    [HttpGet]
    public async Task<IActionResult> GetProducts()
    {
        _logger.LogInformation("Getting all products");

        var products = await _context.Products.ToListAsync();

        _logger.LogInformation("Retrieved {ProductCount} products", products.Count);

        return Ok(products);
    }

    [HttpGet("{id}")]
    public async Task<IActionResult> GetProduct(int id)
    {
        _logger.LogInformation("Getting product with {ProductId}", id);

        var product = await _context.Products.FindAsync(id);

        if (product == null)
        {
            _logger.LogWarning("Product with {ProductId} not found", id);
            return NotFound();
        }

        return Ok(product);
    }

    [HttpPost]
    public async Task<IActionResult> CreateProduct([FromBody] Product product)
    {
        try
        {
            _context.Products.Add(product);
            await _context.SaveChangesAsync();

            _logger.LogInformation(
                "Product created: {ProductId} - {ProductName} - Price: {Price}",
                product.Id, product.Name, product.Price);

            return CreatedAtAction(nameof(GetProduct), new { id = product.Id }, product);
        }
        catch (Exception ex)
        {
            _logger.LogError(ex,
                "Failed to create product {ProductName}",
                product.Name);
            throw;
        }
    }
}

📈 Kibana ile Log Analizi

Adım 1: Kibana'ya Erişim

Tarayıcıda http://localhost:5601 adresine gidin.

Adım 2: Data View (Index Pattern) Oluşturma

  1. Management → Stack Management → Data Views
  2. Create data view butonuna tıklayın
  3. Name: restapi-logs-*
  4. Index pattern: restapi-logs-*
  5. Timestamp field: @timestamp
  6. Save data view

Adım 3: Discover ile Log Keşfi

  1. Analytics → Discover menüsüne gidin
  2. Data view olarak restapi-logs-* seçin
  3. Zaman aralığını ayarlayın (Last 15 minutes)
  4. KQL (Kibana Query Language) ile arama yapın:
# Tüm ERROR seviyesindeki loglar
level: "Error"

# Belirli bir endpoint'e ait loglar
fields.RequestPath: "/api/products"

# HTTP 500 hatalarını bul
fields.StatusCode: 500

# Belirli bir kullanıcının logları
fields.UserId: "user123"

# Birden fazla koşul
level: "Error" AND fields.RequestPath: "/api/products"

# Metin içinde arama
message: "product" AND message: "created"

Adım 4: Visualize ile Görselleştirme

Sık kullanılan Kibana görselleştirmeleri:

  • Log Hacmi (Bar Chart): Zaman bazlı log sayısı — hata spike'larını görme
  • Hata Dağılımı (Pie Chart): Log seviyelerine göre dağılım
  • Response Süreleri (Line Chart): Zaman bazlı ortalama response süresi
  • Top Hatalar (Data Table): En çok tekrarlanan hata mesajları
  • Endpoint Kullanımı (Horizontal Bar): En çok hit alan endpoint'ler

📥 Logstash Pipeline Konfigürasyonu (Opsiyonel)

Serilog'un doğrudan Elasticsearch'e yazması yerine Logstash üzerinden de yönlendirme yapılabilir:

logstash.conf

# logstash.conf
input {
  tcp {
    port => 5000
    codec => json_lines
  }
}

filter {
  # Tarih alanını parse et
  date {
    match => ["@timestamp", "ISO8601"]
  }

  # Log seviyesine göre tag ekle
  if [level] == "Error" or [level] == "Fatal" {
    mutate {
      add_tag => ["error"]
    }
  }

  # Gereksiz alanları kaldır
  mutate {
    remove_field => ["host", "port"]
  }
}

output {
  elasticsearch {
    hosts => ["http://elasticsearch:9200"]
    index => "restapi-logs-%{+YYYY.MM.dd}"
  }

  # Debug: konsola da yazdır
  stdout {
    codec => rubydebug
  }
}
⚠️ Not: Bu konfigürasyon logstash.conf dosyasına yazılır ve docker-compose volume olarak bağlanır. Eğitimde basitlik için Serilog'un doğrudan Elasticsearch'e yazması önerilir.

🧪 Pratik Alıştırma

  1. Serilog paketlerini projeye ekleyin
  2. Program.cs'e Serilog konfigürasyonunu ekleyin
  3. Controller'lara structured logging ekleyin
  4. Uygulamayı çalıştırın ve çeşitli istekler gönderin
  5. Elasticsearch'te index'in oluştuğunu doğrulayın: 
    curl http://localhost:9200/_cat/indices?v
  6. Kibana'da Data View oluşturun ve logları keşfedin
  7. Kasıtlı hata oluşturun (örn: olmayan bir ürün ID'si) ve Kibana'da hata loglarını bulun
