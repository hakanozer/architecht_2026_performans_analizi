🎯 Eğitim Genel Bakış

Bu eğitim serisi, .NET Core REST API uygulamalarında performans analizi, izleme (monitoring), loglama ve metrik toplama konularını kapsamlı bir şekilde ele almaktadır.

💻 Mac & VS Code Ortamına Geçiş

İlk ders Windows üzerinde WinDbg ve Visual Studio ile yürütüldü. Bundan sonraki dersler macOS üzerinde VS Code ile devam edecektir.

VS Code Eklentileri (Extensions)

💡 İpucu: VS Code Extensions panelinden (⌘+Shift+X) aşağıdaki eklentileri yükleyin.
  • C# Dev Kit — .NET geliştirme ortamı
  • .NET Install Tool — .NET SDK yönetimi
  • REST Client — HTTP istekleri doğrudan VS Code'dan gönderme
  • Docker — Container yönetimi
  • YAML — docker-compose ve config dosyaları için
  • Thunder Client — Postman alternatifi
  • GitLens — Git geçmişi ve diff görünümü

macOS'ta .NET SDK Kurulumu

# .NET SDK kurulumu (Homebrew ile)
brew install dotnet-sdk

# Versiyon kontrolü
dotnet --version

# Proje restore
cd RestApi/RestApi
dotnet restore

# Projeyi çalıştırma
dotnet run

Docker Desktop (Mac)

# Docker Desktop yüklü olmalı
# https://www.docker.com/products/docker-desktop/

# Docker versiyon kontrolü
docker --version
docker compose version

# Tüm servisleri başlatma
cd RestApi
docker compose up -d

# Servislerin durumunu kontrol
docker compose ps

Servis Port Amaç URL

✅ Kurulum Doğrulama Adımları

1. Docker Servislerini Başlatma

# Proje dizinine gidin
cd RestApi

# Servisleri başlatın
docker compose up -d

# Durumu kontrol edin
docker compose ps

# Logları izleyin
docker compose logs -f

2. Servis Sağlık Kontrolleri

# Prometheus
curl http://localhost:9090/-/healthy

# Elasticsearch
curl http://localhost:9200/_cluster/health?pretty

# Grafana (varsayılan kullanıcı: admin/admin)
curl http://localhost:3000/api/health

# Kibana
curl http://localhost:5601/api/status

# Jaeger
curl http://localhost:16686/

3. .NET Uygulamasını Başlatma

cd RestApi/RestApi
dotnet restore
dotnet run
✅ Başarılı: Tüm servisler çalışıyorsa ve health check'ler geçiyorsa, eğitim ortamınız hazırdır!
