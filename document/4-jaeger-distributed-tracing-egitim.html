📖 Jaeger Nedir?

Jaeger, Uber tarafından geliştirilen ve CNCF'e bağışlanan açık kaynak bir distributed tracing platformudur. Mikroservis mimarisinde isteklerin servisler arasındaki yolculuğunu görselleştirmeye yarar.

Jaeger ile Neler Yapılabilir?

  • Performans darboğaz tespiti: Hangi servis veya işlem en fazla süre harcıyor?
  • Hata analizi: Hangi serviste hata oluşuyor?
  • Bağımlılık grafiği: Servisler arası iletişim haritası
  • Latency optimizasyonu: Yavaş endpoint'lerin tespiti
  • Root cause analysis: Sorunun kaynağını belirleme

🖥️ Jaeger UI Kullanım Rehberi

UI'a Erişim

Tarayıcıda http://localhost:16686 adresini açın.

Trace Arama

  1. Service dropdown'ından uygulamanızı seçin (örn: "RestApi")
  2. Operation ile belirli bir endpoint filtreleyin (örn: "GET /api/products")
  3. Tags ile özel filtre ekleyin (örn: http.status_code=500)
  4. Lookback ile zaman aralığını belirleyin
  5. Min/Max Duration ile süre bazlı filtreleme yapın
  6. Find Traces butonuna tıklayın

Trace Detay İnceleme

Bir trace'e tıkladığınızda göreceğiniz bilgiler:

  • Timeline görünümü: Her span'in ne kadar süre harcadığını gösteren Gantt chart
  • Span detayları: Tags, logs, process bilgileri
  • Critical path: İsteğin toplam süresini belirleyen en uzun yol
  • Span ilişkileri: Parent-child span hiyerarşisi

📊 Trace Karşılaştırma ve Analiz

İki Trace'i Karşılaştırma

Jaeger UI'da iki trace'i yan yana karşılaştırabilirsiniz. Bu, performans regresyonlarını tespit etmek için çok kullanışlıdır:

  1. İki trace seçin (checkbox ile)
  2. Compare butonuna tıklayın
  3. Span süreleri arasındaki farkları inceleyin

DAG (Dependency Graph) Görünümü

System Architecture sekmesinde servisler arası bağımlılık grafiğini görüntüleyebilirsiniz. Bu grafik trace verilerinden otomatik oluşturulur.

Performans Analizi Soruları

  • En yavaş endpoint hangisi? (Operation bazlı sıralama)
  • Veritabanı sorguları toplam sürenin yüzde kaçını kaplıyor?
  • Servisler arası iletişimde en fazla gecikme nerede?
  • Hatalı trace'lerde ortak olan span hangisi?
  • p95 ve p99 süreleri arasında büyük fark var mı?

🛠️ Zengin Trace Verisi Oluşturma

İş Mantığı Span'leri

using System.Diagnostics;

public class ProductService
{
    private static readonly ActivitySource ActivitySource =
        new("RestApi.ProductService");

    private readonly ApplicationDbContext _context;
    private readonly ILogger<ProductService> _logger;

    public ProductService(ApplicationDbContext context, ILogger<ProductService> logger)
    {
        _context = context;
        _logger = logger;
    }

    public async Task<List<Product>> GetAllProductsAsync()
    {
        using var activity = ActivitySource.StartActivity("GetAllProducts");

        // Validation span
        using (var validationActivity = ActivitySource.StartActivity("ValidateRequest"))
        {
            validationActivity?.SetTag("validation.type", "basic");
            // Validation logic...
            await Task.Delay(2); // Simüle
        }

        // Cache kontrol span
        using (var cacheActivity = ActivitySource.StartActivity("CheckCache"))
        {
            cacheActivity?.SetTag("cache.type", "memory");
            cacheActivity?.SetTag("cache.hit", false);
            // Cache miss simülasyonu
        }

        // Veritabanı sorgusu span
        List<Product> products;
        using (var dbActivity = ActivitySource.StartActivity("DatabaseQuery"))
        {
            dbActivity?.SetTag("db.system", "sqlite");
            dbActivity?.SetTag("db.name", "restapi");
            dbActivity?.SetTag("db.operation", "SELECT");
            dbActivity?.SetTag("db.statement", "SELECT * FROM Products");

            products = await _context.Products.ToListAsync();

            dbActivity?.SetTag("db.result_count", products.Count);
        }

        activity?.SetTag("product.count", products.Count);
        activity?.AddEvent(new ActivityEvent("ProductsLoaded",
            tags: new ActivityTagsCollection
            {
                { "count", products.Count }
            }));

        return products;
    }
}

Hata Senaryosu Span'i

public async Task<Product> GetProductByIdAsync(int id)
{
    using var activity = ActivitySource.StartActivity("GetProductById");
    activity?.SetTag("product.id", id);

    try
    {
        var product = await _context.Products.FindAsync(id);

        if (product == null)
        {
            activity?.SetStatus(ActivityStatusCode.Error, $"Product {id} not found");
            activity?.AddEvent(new ActivityEvent("ProductNotFound",
                tags: new ActivityTagsCollection { { "product.id", id } }));
            throw new KeyNotFoundException($"Product with ID {id} not found");
        }

        activity?.SetStatus(ActivityStatusCode.Ok);
        return product;
    }
    catch (Exception ex)
    {
        activity?.SetStatus(ActivityStatusCode.Error, ex.Message);
        activity?.RecordException(ex);
        throw;
    }
}

🧪 Pratik Alıştırma

  1. Docker servislerin çalıştığını doğrulayın (docker compose ps)
  2. .NET uygulamasını OpenTelemetry ile çalıştırın
  3. API'ye çeşitli istekler gönderin: 
    # Başarılı istekler
curl http://localhost:5000/api/products
curl http://localhost:5000/api/products/1

# Hata oluşturacak istekler
curl http://localhost:5000/api/products/99999
curl -X POST http://localhost:5000/api/products \
  -H "Content-Type: application/json" \
  -d '{}'
  4. Jaeger UI'da (http://localhost:16686) trace'leri inceleyin
  5. Başarılı ve hatalı trace'leri karşılaştırın
  6. En yavaş span'i tespit edin
  7. System Architecture grafiğini inceleyin
🎯 Hedef: Jaeger UI'da bir hata trace'ini bulup, hatanın hangi span'de (işlemde) oluştuğunu tespit edebilmelisiniz.
