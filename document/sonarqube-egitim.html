🎯 SonarQube Nedir?

SonarQube, yazılım projelerinde statik kod analizi yaparak kod kalitesini ölçen, güvenlik açıklarını tespit eden ve teknik borcu yöneten açık kaynaklı bir sürekli inceleme (Continuous Inspection) platformudur.

🐳 Docker Compose ile Kurulum

Projemizdeki docker-compose.yml dosyasında SonarQube aşağıdaki şekilde yapılandırılmıştır:

# docker-compose.yml — SonarQube Servisleri

# ✅ SONARQUBE
sonarqube:
  image: sonarqube:community
  container_name: sonarqube
  depends_on:
    - sonarqube_db
  ports:
    - "9000:9000"
  environment:
    SONAR_JDBC_URL: jdbc:postgresql://sonarqube_db:5432/sonar
    SONAR_JDBC_USERNAME: sonar
    SONAR_JDBC_PASSWORD: sonar
  volumes:
    - sonarqube_data:/opt/sonarqube/data
    - sonarqube_logs:/opt/sonarqube/logs
    - sonarqube_extensions:/opt/sonarqube/extensions
  restart: unless-stopped

# ✅ SONAR DB
sonarqube_db:
  image: postgres:13
  container_name: sonarqube_db
  environment:
    POSTGRES_USER: sonar
    POSTGRES_PASSWORD: sonar
    POSTGRES_DB: sonar
  volumes:
    - sonarqube_postgres:/var/lib/postgresql/data
  restart: unless-stopped

volumes:
  sonarqube_data:
  sonarqube_logs:
  sonarqube_extensions:
  sonarqube_postgres:

1 Servisleri Başlatma

# Tüm servisleri başlat
docker-compose up -d sonarqube sonarqube_db

# Logları kontrol et
docker-compose logs -f sonarqube

# Servis durumunu kontrol et
docker-compose ps

2 İlk Erişim

⚠️ Önemli: SonarQube'un tamamen başlaması 1-2 dakika sürebilir. Tarayıcıdan http://localhost:9000 adresine gidin. Varsayılan giriş bilgileri:
Kullanıcı adı: admin | Şifre: admin
İlk girişte şifre değişikliği istenecektir.

⚙️ .NET SonarScanner Kurulumu

.NET projelerini analiz etmek için dotnet-sonarscanner global aracını kullanıyoruz.

1 SonarScanner Kurulumu

# Global tool olarak yükleme
dotnet tool install --global dotnet-sonarscanner

# Kurulumu doğrulama
dotnet sonarscanner --version

2 SonarQube'da Proje Oluşturma

  1. http://localhost:9000 adresine gidin
  2. Projects → Create Project → Manually seçin
  3. Project Key: restapi girin
  4. Display Name: RestApi girin
  5. Set Up butonuna tıklayın
  6. Locally seçeneğini seçin
  7. Bir Token oluşturun ve kaydedin

3 Analiz Komutları

# Adım 1: Analizi başlat
dotnet sonarscanner begin \
  /k:"restapi" \
  /d:sonar.host.url="http://localhost:9000" \
  /d:sonar.token="BURAYA_TOKEN_YAZIN"

# Adım 2: Projeyi derle
dotnet build RestApi.sln

# Adım 3: Analizi tamamla ve sonuçları gönder
dotnet sonarscanner end \
  /d:sonar.token="BURAYA_TOKEN_YAZIN"
✅ İpucu: Analiz tamamlandıktan sonra sonuçları http://localhost:9000/dashboard?id=restapi adresinde görebilirsiniz.

🚦 Kalite Kapıları (Quality Gates)

Quality Gate, projenin belirli kalite standartlarını karşılayıp karşılamadığını kontrol eden otomatik bir eşik mekanizmasıdır. Varsayılan olarak "Sonar way" kalite kapısı aktiftir.

Varsayılan Sonar Way Koşulları

Metrik Koşul Eşik Değer
Yeni Kod — Güvenlik Açıkları Daha az olmalı 0
Yeni Kod — Hatalar (Bugs) Daha az olmalı 0
Yeni Kod — Code Smells (Oranı) A olmalı A (≤5%)
Yeni Kod — Kapsam (Coverage) Daha büyük olmalı ≥80%
Yeni Kod — Tekrarlanan Satırlar Daha az olmalı ≤3%

Özel Quality Gate Oluşturma

  1. Quality Gates menüsüne gidin
  2. Create butonuna tıklayın
  3. İsim verin: örn. Architecht Quality Gate
  4. Add Condition ile özel koşullar ekleyin
  5. Projenize atayın
💡 Quality Gate Durumları:
🟢 Passed — Tüm koşullar sağlandı
🔴 Failed — En az bir koşul sağlanamadı

📏 Kural Profilleri (Quality Profiles)

Quality Profile, projenize uygulanacak analiz kurallarının koleksiyonudur. Her dil için ayrı profil tanımlanır.

C# İçin Varsayılan Kurallar

Kural Kategorisi Açıklama Örnek
Bug Çalışma zamanında hata oluşturabilecek kodlar Null referans kontrolü eksik
Vulnerability Güvenlik açığı oluşturabilecek kodlar SQL Injection riski, hardcoded şifreler
Code Smell Bakımı zorlaştıran, temiz kod ilkelerine uymayan kodlar Fazla uzun metotlar, kullanılmayan değişkenler
Security Hotspot Güvenlik açısından incelenmesi gereken kodlar Kriptografi kullanımı, CORS yapılandırması

Profil Özelleştirme

# Quality Profiles → C# → Sonar way → Copy

# Özel profilde kural etkinleştirme/devre dışı bırakma:
# 1. Rules menüsünde kural arayın
# 2. Activate/Deactivate butonunu kullanın
# 3. Severity (önem derecesi) ayarlayın:
#    - Blocker  → Kesinlikle düzeltilmeli
#    - Critical → Kritik öncelikli
#    - Major    → Yüksek öncelikli
#    - Minor    → Düşük öncelikli

🛠️ RestApi Projesi Üzerinde Analiz

Mevcut RestApi projemiz üzerinde SonarQube analizi yapıldığında aşağıdaki türde bulgular elde edilmektedir:

Örnek Bulgular (CS8618 — Null Atanamaz Özellik)

// ⚠️ SonarQube Bulgusu: CS8618
// Dosya: Data/ApplicationDbContext.cs

// SORUN: Null atanamaz özellik, oluşturucudan çıkış yaparken
// null olmayan bir değer içermelidir.
public class ApplicationDbContext : DbContext
{
    // ❌ Bu özellikler null olabilir uyarısı verir
    public DbSet<User> Users { get; set; }
    public DbSet<Product> Products { get; set; }
    public DbSet<Note> Notes { get; set; }

    // ✅ Düzeltme seçenekleri:
    // Seçenek 1: = null! ile başlatma
    public DbSet<User> Users { get; set; } = null!;

    // Seçenek 2: Nullable yapma
    public DbSet<User>? Users { get; set; }

    // Seçenek 3: Constructor'da başlatma
    public ApplicationDbContext(DbContextOptions options) : base(options)
    {
        Users = Set<User>();
    }
}

Güvenlik Analizi — Kontrol Edilmesi Gerekenler

Dosya Konu Öneri
AuthController.cs JWT Token yönetimi Token süresini ve secret key yönetimini kontrol edin
PasswordManager.cs Şifre hashleme Güvenli hashing algoritması kullanıldığını doğrulayın
Program.cs CORS ve Middleware yapılandırması Üretim ortamı için CORS politikasını kısıtlayın
appsettings.json Hassas bilgi yönetimi Connection string ve secret'ları environment variable ile yönetin
🔴 Dikkat: SonarQube, appsettings.json dosyasındaki hardcoded değerleri güvenlik açığı olarak işaretleyebilir. Üretim ortamında bu değerleri environment variable veya Azure Key Vault gibi bir secret yönetim aracı ile yönetin.

🔄 CI/CD Pipeline Entegrasyonu

SonarQube analizi CI/CD pipeline'ına entegre edilerek her commit veya pull request için otomatik kalite kontrolü sağlanabilir.

GitHub Actions Entegrasyonu

# .github/workflows/sonarqube.yml
name: SonarQube Analysis

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  sonarqube:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Setup .NET
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: '9.0.x'

      - name: Install SonarScanner
        run: dotnet tool install --global dotnet-sonarscanner

      - name: Begin Analysis
        run: |
          dotnet sonarscanner begin \
            /k:"restapi" \
            /d:sonar.host.url="${{ secrets.SONAR_HOST_URL }}" \
            /d:sonar.token="${{ secrets.SONAR_TOKEN }}"

      - name: Build
        run: dotnet build RestApi/RestApi.sln --no-incremental

      - name: End Analysis
        run: |
          dotnet sonarscanner end \
            /d:sonar.token="${{ secrets.SONAR_TOKEN }}"
💡 İpucu: GitHub Secrets içine SONAR_HOST_URL ve SONAR_TOKEN değerlerini ekleyin. Lokal geliştirme için http://localhost:9000 kullanabilirsiniz.

💻 SonarLint — IDE Entegrasyonu

SonarLint, SonarQube kurallarını doğrudan IDE'nizde çalıştıran bir eklentidir. Kodu yazarken anında geri bildirim sağlar.

VS Code İçin Kurulum

  1. VS Code Extensions panelinden SonarLint aratın
  2. SonarLint (by Sonarsource) eklentisini yükleyin
  3. VS Code ayarlarında Connected Mode yapılandırın:
// .vscode/settings.json
{
    "sonarlint.connectedMode.connections.sonarqube": [
        {
            "connectionId": "local-sonarqube",
            "serverUrl": "http://localhost:9000",
            "token": "BURAYA_TOKEN_YAZIN"
        }
    ],
    "sonarlint.connectedMode.project": {
        "connectionId": "local-sonarqube",
        "projectKey": "restapi"
    }
}

Visual Studio İçin Kurulum

  1. Extensions → Manage Extensions menüsüne gidin
  2. SonarLint for Visual Studio aratın ve yükleyin
  3. Analyze → Manage SonarQube Connections üzerinden bağlantı kurun
  4. Proje binding yaparak kuralları senkronize edin
✅ Avantaj: SonarLint ile kodunuzu SonarQube sunucusuna göndermeden önce yerel ortamınızda analiz edebilir, sorunları erkenden tespit edebilirsiniz. Bu yaklaşım "Shift Left" prensibini uygular.

🧪 Uygulamalı Egzersizler

Egzersiz 1: İlk SonarQube Analizi

  1. Docker Compose ile SonarQube ve PostgreSQL servislerini başlatın
  2. SonarQube web arayüzüne giriş yapın (http://localhost:9000)
  3. Yeni bir proje oluşturun (Project Key: restapi)
  4. Token oluşturun
  5. dotnet-sonarscanner ile analiz çalıştırın
  6. Dashboard'daki sonuçları inceleyin

Egzersiz 2: Bulguları Düzeltme

  1. SonarQube dashboard'da bulunan Bugs sekmesine gidin
  2. En az 2 bulguyu düzeltin
  3. Analizi tekrar çalıştırın
  4. Düzeltmelerin sonuçlara yansıdığını doğrulayın

Egzersiz 3: Özel Quality Gate

  1. Yeni bir Quality Gate oluşturun: Architecht QG
  2. Aşağıdaki koşulları ekleyin:
    • Yeni kod coverage ≥ 70%
    • Yeni Bug sayısı = 0
    • Yeni Vulnerability sayısı = 0
    • Tekrarlanan satır oranı ≤ 5%
  3. Quality Gate'i projenize atayın
  4. Analizi tekrar çalıştırarak Quality Gate durumunu kontrol edin

Egzersiz 4: SonarLint Entegrasyonu

  1. VS Code veya Visual Studio'ya SonarLint eklentisini yükleyin
  2. Connected Mode ile SonarQube sunucunuza bağlanın
  3. Bir dosyayı açın ve SonarLint uyarılarını inceleyin
  4. Uyarıları düzeltin ve SonarLint geri bildirimini gözlemleyin

✅ En İyi Uygulamalar (Best Practices)

Genel Öneriler

  • Clean as You Code: Yeni kod her zaman kalite standartlarına uygun olmalıdır. Mevcut teknik borcu aşamalı olarak azaltın.
  • Quality Gate Zorunlu Kılın: CI/CD pipeline'ında Quality Gate başarısız olursa merge işlemi engellenmelidir.
  • SonarLint Kullanın: IDE'de anlık geri bildirim alarak sorunları erkenden tespit edin.
  • Kuralları Özelleştirin: Projenize uygun olmayan kuralları devre dışı bırakın, önemli kuralların severity'sini artırın.
  • Düzenli Analiz: Her commit'te otomatik analiz çalıştırın.

Güvenlik Önerileri

  • SonarQube token'larını environment variable olarak saklayın, kaynak koda eklemeyin
  • Security Hotspot'ları mutlaka inceleyin ve "Safe" veya "Fixed" olarak işaretleyin
  • Üretim ortamında SonarQube'a erişimi yetkilendirme ile kısıtlayın
  • OWASP Top 10 kurallarını aktif tutun
⚠️ Hatırlatma: SonarQube, tüm güvenlik sorunlarını yakalayamaz. Statik analiz, dinamik test (DAST) ve penetrasyon testleri ile birlikte kullanılmalıdır. Bu, "Defense in Depth" (derinlemesine savunma) stratejisinin bir parçasıdır.

🔧 Sorun Giderme (Troubleshooting)

Sık Karşılaşılan Sorunlar

Sorun Çözüm
SonarQube başlamıyor vm.max_map_count değerini artırın:
sysctl -w vm.max_map_count=524288
Analiz bağlantı hatası veriyor SonarQube'un tamamen başladığından emin olun. Logları kontrol edin:
docker-compose logs sonarqube
Token geçersiz hatası SonarQube arayüzünden yeni token oluşturun:
My Account → Security → Generate Tokens
Analiz sonuçları görünmüyor dotnet sonarscanner end komutunun başarıyla tamamlandığından emin olun. Background task durumunu kontrol edin:
Administration → Background Tasks
PostgreSQL bağlantı hatası sonarqube_db servisinin çalıştığından emin olun:
docker-compose ps sonarqube_db
