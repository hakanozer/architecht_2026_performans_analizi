🔥 Prometheus Nedir?

Prometheus, açık kaynak bir zaman serisi veritabanı ve monitoring sistemidir. Metrikleri pull modeli ile hedef uygulamalardan toplar ve güçlü bir sorgulama dili (PromQL) sunar.

Temel Özellikler

  • Pull-based model: Prometheus hedef uygulamaların /metrics endpoint'ini belirli aralıklarla çeker
  • PromQL: Güçlü ve esnek sorgulama dili
  • Multi-dimensional data: Label tabanlı metrik modeli
  • Service discovery: Otomatik hedef keşfi
  • Alerting: AlertManager ile entegre uyarı sistemi

📏 Prometheus Metrik Tipleri

Tip Açıklama Kullanım Alanı Örnek
Counter Sadece artan sayaç Toplam istek sayısı, hata sayısı http_requests_total
Gauge Artıp azalan değer Sıcaklık, bellek kullanımı, aktif bağlantı sayısı process_memory_bytes
Histogram Dağılım ölçümü (bucket'lar) Response süresi dağılımı, boyut dağılımı http_request_duration_seconds
Summary İstemci tarafı quantile hesaplaması Percentile hesaplamaları (p50, p95, p99) rpc_duration_seconds

⚙️ .NET Core'da Prometheus Metrikleri

OpenTelemetry ile Prometheus Exporter

OpenTelemetry modülünde eklediğimiz Prometheus exporter'ı sayesinde /metrics endpoint'i otomatik oluşturulur:

// Program.cs — Metrics konfigürasyonu (OpenTelemetry ile)
builder.Services.AddOpenTelemetry()
    .WithMetrics(metrics =>
    {
        metrics
            .SetResourceBuilder(resourceBuilder)
            // ASP.NET Core metrikleri (request duration, request count vb.)
            .AddAspNetCoreInstrumentation()
            // HttpClient metrikleri (outgoing request'ler)
            .AddHttpClientInstrumentation()
            // .NET Runtime metrikleri (GC, ThreadPool, Memory)
            .AddRuntimeInstrumentation()
            // Prometheus scraping endpoint'i
            .AddPrometheusExporter();
    });

// ...
var app = builder.Build();

// Prometheus scraping endpoint'ini aktifleştir
// Bu, /metrics endpoint'ini oluşturur
app.MapPrometheusScrapingEndpoint();

Özel (Custom) Metrikler Tanımlama

using System.Diagnostics.Metrics;

public class ProductMetrics
{
    private readonly Counter<long> _productCreatedCount;
    private readonly Histogram<double> _productQueryDuration;
    private readonly UpDownCounter<int> _activeRequests;

    public ProductMetrics(IMeterFactory meterFactory)
    {
        var meter = meterFactory.Create("RestApi.Products");

        // Counter: Toplam oluşturulan ürün sayısı
        _productCreatedCount = meter.CreateCounter<long>(
            "products.created.total",
            description: "Toplam oluşturulan ürün sayısı");

        // Histogram: Ürün sorgu süresi
        _productQueryDuration = meter.CreateHistogram<double>(
            "products.query.duration",
            unit: "ms",
            description: "Ürün sorgulama süresi (ms)");

        // UpDownCounter: Aktif istek sayısı
        _activeRequests = meter.CreateUpDownCounter<int>(
            "products.active_requests",
            description: "Aktif istek sayısı");
    }

    public void RecordProductCreated() => _productCreatedCount.Add(1);

    public void RecordQueryDuration(double durationMs) =>
        _productQueryDuration.Record(durationMs);

    public void IncrementActiveRequests() => _activeRequests.Add(1);
    public void DecrementActiveRequests() => _activeRequests.Add(-1);
}

// DI'a kaydet
builder.Services.AddSingleton<ProductMetrics>();

// OpenTelemetry'ye custom meter'ı ekle
builder.Services.AddOpenTelemetry()
    .WithMetrics(metrics =>
    {
        metrics
            .AddMeter("RestApi.Products")  // Custom meter kaydı
            // ... diğer ayarlar
    });

Controller'da Kullanım

[ApiController]
[Route("api/[controller]")]
public class ProductController : ControllerBase
{
    private readonly ProductMetrics _metrics;
    private readonly ApplicationDbContext _context;

    public ProductController(ApplicationDbContext context, ProductMetrics metrics)
    {
        _context = context;
        _metrics = metrics;
    }

    [HttpGet]
    public async Task<IActionResult> GetProducts()
    {
        _metrics.IncrementActiveRequests();
        var sw = Stopwatch.StartNew();

        try
        {
            var products = await _context.Products.ToListAsync();
            return Ok(products);
        }
        finally
        {
            sw.Stop();
            _metrics.RecordQueryDuration(sw.ElapsedMilliseconds);
            _metrics.DecrementActiveRequests();
        }
    }

    [HttpPost]
    public async Task<IActionResult> CreateProduct([FromBody] Product product)
    {
        _context.Products.Add(product);
        await _context.SaveChangesAsync();

        _metrics.RecordProductCreated();  // Metrik kaydet

        return CreatedAtAction(nameof(GetProducts), new { id = product.Id }, product);
    }
}

📝 Prometheus Konfigürasyonu

Mevcut prometheus.yml dosyasını .NET uygulamasını da scrape edecek şekilde güncelleyin:

# prometheus.yml
global:
  scrape_interval: 5s          # Her 5 saniyede bir metrik topla
  evaluation_interval: 5s       # Alert kurallarını her 5 saniyede değerlendir

scrape_configs:
  # Prometheus kendi metriklerini toplar
  - job_name: 'prometheus'
    static_configs:
      - targets: ['prometheus:9090']

  # .NET REST API metrikleri
  - job_name: 'restapi'
    # Mac'te docker dışından çalışan .NET uygulaması
    static_configs:
      - targets: ['host.docker.internal:5000']
    # Metrics endpoint yolu
    metrics_path: '/metrics'
    scheme: 'http'
⚠️ Mac Docker Notu: macOS'ta Docker container'lardan host makinedeki uygulamaya erişmek için host.docker.internal kullanılır. Linux'ta 172.17.0.1 veya --network=host kullanılabilir.

Doğrulama

# .NET uygulamasının metrics endpoint'ini kontrol
curl http://localhost:5000/metrics

# Prometheus targets sayfasını kontrol
# http://localhost:9090/targets
# "restapi" target'ının UP durumda olduğunu doğrulayın

🔍 PromQL Sorguları

Prometheus'un sorgulama arayüzünden (http://localhost:9090) aşağıdaki sorguları deneyin:

Temel Sorgular

# Toplam HTTP istek sayısı
http_server_request_duration_seconds_count

# Son 5 dakikadaki istek oranı (rate)
rate(http_server_request_duration_seconds_count[5m])

# HTTP metod bazında istek oranı
rate(http_server_request_duration_seconds_count{http_request_method="GET"}[5m])

# Ortalama response süresi
rate(http_server_request_duration_seconds_sum[5m])
/
rate(http_server_request_duration_seconds_count[5m])

# 95. percentile response süresi
histogram_quantile(0.95, rate(http_server_request_duration_seconds_bucket[5m]))

# 99. percentile response süresi
histogram_quantile(0.99, rate(http_server_request_duration_seconds_bucket[5m]))

Runtime Metrikleri

# .NET process bellek kullanımı
process_runtime_dotnet_gc_heap_size_bytes

# GC collection sayısı
process_runtime_dotnet_gc_collections_count

# ThreadPool aktif thread sayısı
process_runtime_dotnet_thread_pool_threads_count

Custom Metrik Sorguları

# Toplam oluşturulan ürün sayısı
products_created_total

# Son 1 dakikadaki ortalama sorgu süresi
rate(products_query_duration_sum[1m]) / rate(products_query_duration_count[1m])

# Aktif istek sayısı
products_active_requests

📊 Grafana Dashboard Oluşturma

Adım 1: Grafana'ya Giriş

  1. Tarayıcıda http://localhost:3000 açın
  2. Varsayılan kullanıcı: admin / admin
  3. İlk girişte şifre değişikliği istenir (Skip diyebilirsiniz)

Adım 2: Prometheus Data Source Ekleme

  1. Configuration → Data Sources → Add data source
  2. Prometheus seçin
  3. URL: http://prometheus:9090
  4. Save & Test butonuna tıklayın
💡 İpucu: Grafana ve Prometheus aynı Docker network'te olduğu için, container adı (prometheus) doğrudan kullanılabilir.

Adım 3: Dashboard Oluşturma

  1. Dashboards → New Dashboard → Add visualization
  2. Data source olarak Prometheus seçin
  3. Aşağıdaki panelleri ekleyin:

Panel 1: HTTP İstek Oranı (Time Series)

# PromQL
rate(http_server_request_duration_seconds_count[1m])

# Legend: {{http_request_method}} {{http_route}}
# Title: HTTP İstek Oranı (req/s)

Panel 2: Response Süresi — Percentile (Time Series)

# p50 Query
histogram_quantile(0.5, rate(http_server_request_duration_seconds_bucket[5m]))

# p95 Query
histogram_quantile(0.95, rate(http_server_request_duration_seconds_bucket[5m]))

# p99 Query
histogram_quantile(0.99, rate(http_server_request_duration_seconds_bucket[5m]))

# Title: Response Süresi Percentile (sn)

Panel 3: Bellek Kullanımı (Gauge)

# PromQL
process_runtime_dotnet_gc_heap_size_bytes / 1024 / 1024

# Unit: megabytes
# Title: .NET Heap Bellek (MB)

Panel 4: HTTP Durum Kodları (Pie Chart)

# PromQL
sum by(http_response_status_code)(
  increase(http_server_request_duration_seconds_count[5m])
)

# Title: HTTP Durum Kodu Dağılımı

🧪 Pratik: Yük Testi ile Metrik Gözlemleme

Basit Yük Testi (Terminal'den)

# 100 ardışık istek gönder
for i in $(seq 1 100); do
  curl -s -o /dev/null -w "%{http_code}\n" http://localhost:5000/api/products
done

# Paralel istek gönder (xargs ile)
seq 1 50 | xargs -P 10 -I {} curl -s -o /dev/null -w "%{http_code}\n" \
  http://localhost:5000/api/products

# hey aracı ile yük testi (brew install hey)
hey -n 1000 -c 10 http://localhost:5000/api/products
✅ Gözlem: Yük testi sırasında Grafana dashboard'unu açık tutun. İstek oranı, response süresi ve bellek kullanımı grafiklerindeki değişimleri gerçek zamanlı izleyin.
